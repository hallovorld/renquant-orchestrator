# expkit — the reusable pre-registered experiment framework

STATUS: implemented in this PR (`src/renquant_orchestrator/expkit/`), library + tests only —
no production behavior change, no config change, reads production data nowhere (plugins read
committed evidence or declared read-only substrates).
DATE: 2026-07-03
RELATION: mechanizes the S-REL reliability contract
(`doc/design/2026-07-03-s-rel-experiment-reliability.md` R1–R4 + §3.1) as an importable
library. S-REL §3.3 deferred "automated manifest generation helpers" until the ACTIVE queue
resolved — V1–V3 have resolved (UPHELD/UPHELD/WEAKENED), and the 07-02/03 measurement burst
wrote ~8,800 lines of per-experiment scripts re-implementing the same machinery 6+ times.
This is that revisit.

## 1. Problem

Every experiment of the burst (C2/C3/C4, M3, M8, S9, RS-5, D3) is a 550–1,100-line script
that re-implements: per-date Spearman IC, the shifted-label placebo, the carried-mask block
bootstrap, Bonferroni one-sided bounds, seed loops, positive/null controls, evidence
manifests, and a frozen-rule verdict. The copies have drifted (three placebo-shift
conventions, two bootstrap namings, ad-hoc manifest fields), and each new experiment re-pays
the full implementation-bug risk the S-REL verifications exist to catch — V3's WEAKENED
finding and the λ-round-1 precedent were both machinery bugs, not judgment errors.

## 2. Shape

An experiment becomes a `FrozenSpec` + four callbacks; everything else is the library:

```
prereg.py      freeze-first: FrozenSpec dataclass (hypothesis, frozen Criterion thresholds,
               Bonferroni family k, seeds, R3 evidence-boundary fields — validated present,
               R4 reopening conditions — validated non-empty), canonical-JSON sha256, and the
               git-history-aware check that the spec file was COMMITTED strictly before any
               results file (the M8/D3/C2 three-commit pattern, mechanized).
evaluation.py  placebo-difference primitives: per-date cross-sectional Spearman IC (C3
               semantics, imported by C2, re-implemented by C4/RS-5 — now one impl);
               fwd-excess labels (clip ±0.5); BOTH frozen placebo-shift conventions
               (shift=horizon, and gate_shift_sessions = 2×horizon — the repaired-WF-gate /
               S2 convention); paired per-date deltas (D3); matched-admission-rate solver
               (the M4-b matched-breadth protocol).
stats.py       carried-mask moving-block bootstrap (bit-identical to the c3 machinery — the
               C2 regression fixture proves it) + the AUTOMATIC small-n branch: usable
               blocks < 4 (V3 method note: n_dates/block < ~4) ⇒ the bootstrap is REFUSED,
               exact tail masses replace it, and `requires_null_control` is stamped; exact
               enumeration (M3-verification method) with an exact sign-test fallback;
               multi-seed unanimity (#264: seeds are robustness, never extra looks).
controls.py    mandatory first-class controls: positive plant (declared effect size,
               detection asserted) and true null (sign-flip / within-date permutation /
               offset-seed, non-detection + negative-branch-fires asserted; waivable only
               with a documented reason — the C2 PC-C precedent). Injection helpers for
               both IC-series-level and score-level plants/nulls.
evidence.py    S-REL §3.1 manifests: input content-sha256s, code git SHA (worktree-safe) +
               dirty flag, spec/config hashes, seeds, optional env lock; verifying loader
               that recomputes input hashes on re-read and refuses drifted evidence.
verdict.py     GO / KILL / NULL / INCONCLUSIVE / NON-VOTING. NULL-vs-INCONCLUSIVE is
               MECHANICAL (powered iff z·SE_max ≤ bar; the MDE is always reported next to
               the observed effect); the small-n exact branch can never produce NULL;
               NON-VOTING overlays the mechanical output when controls are missing/failed
               or a declared substrate disqualifier exists (the C2 EXPLORATORY_NON_VOTING
               pattern) — the mechanical output is still computed and recorded. Emits
               VERDICTS.md ledger rows in the seeded format.
runner.py      the contract order: freeze check → controls gate → evaluation → stats →
               verdict → hash-stamped evidence bundle. `--dry-run` validates wiring/spec
               and returns the plan without loading substrate or writing anything. The
               controls gate is not skippable: no passing controls ⇒ NON-VOTING, never a
               verdict.
plugins/       one module per experiment. Resident: `c2_quality` (the migration proof).
```

## 3. Migration proof (C2, #275)

`plugins/c2_quality.py` restates C2's frozen constants as a `FrozenSpec` (asserted 1:1
against the committed `frozen_thresholds` block) and replays the committed per-date evidence
through the generic runner. `tests/test_expkit_c2_regression.py` proves against
`doc/research/evidence/2026-07-03-c2/c2_results.json`:

- all three seeds' bootstrap LB/UB/SE/CI95 reproduce (identical resample draws; agreement
  ~1e-13, asserted at 1e-9 — the per-date JSON's serialization precision);
- mean real/placebo/clean IC, hit rate, n=2,241 reproduce;
- the mechanical rule output reproduces (INCONCLUSIVE) and the governing adjudication
  reproduces (NON-VOTING via C2's five declared substrate disqualifiers);
- the freeze-first check passes against the REAL history (the addendum commit is an
  ancestor of the results commit).

Replay-mode controls are IC-series-level wiring checks (mean-shift plant at ~1.25× the
harness's own MDE; offset-seed sign-flip null with the PC-C "specificity only" waiver,
documented) — the original score-level PC-A/PC-B/PC-C remain the committed controls of
record for the original run. The heavy full-substrate path stays in
`scripts/msig_c2_quality.py`; the plugin proves the library reproduces the adjudication
pipeline, it does not fork the generator of record.

## 4. One bug found by writing the tests

The two small-n exact methods define their raw tail masses with OPPOSITE GO/KILL
orientation (bootstrap resample-mean tail mass vs null-hypothesis binomial p-value). The
first draft of the verdict layer read one convention for both — a planted positive effect
under the sign-test fallback would have mapped to KILL. Fixed by explicit
`go_evidence_p`/`kill_evidence_p` fields the verdict layer must read instead of the raw
masses, plus a regression test pinning the orientation agreement. (The framework's own
positive-control discipline is what surfaced this — the λ-round-1 lesson applying to the
library that codifies it.)

## 5. Non-goals / boundaries

- NOT a mass migration: existing merged scripts stay the committed generators of their
  evidence; new experiments start on expkit; old ones migrate only when touched.
- NO production surface: nothing in daily orchestration imports expkit.
- NOT a CI compliance linter for memos (S-REL §3.3 still defers that).
- Substrate loading stays plugin-side and read-only; the library never writes outside the
  caller's `--out`.

## 6. Usage sketch

```python
from renquant_orchestrator.expkit import Criterion, FrozenSpec
from renquant_orchestrator.expkit.runner import ExperimentPlugin, run_experiment

spec = FrozenSpec(experiment_id="my_exp", hypothesis="...",
                  criteria=(Criterion("placebo_clean_ic", 0.015),),
                  family_size_k=3, seeds=(42, 43, 44),
                  evidence_boundary={...7 R3 fields...},
                  reopening_conditions=("...",))
plugin = ExperimentPlugin(spec=spec, load_substrate=..., evaluate=...,
                          positive_control=..., null_control=...,
                          spec_path=..., results_paths=(...,))
result = run_experiment(plugin, out_dir="doc/research/evidence/<date>-<id>",
                        repo_root=".", dry_run=False)
print(result.outcome, result.decision.mde, result.evidence_path)
print(result.verdicts_md_row(date="2026-07-03", rationale="..."))
```
