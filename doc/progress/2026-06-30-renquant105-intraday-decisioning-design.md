New **§9.2d**: the execution-quality estimand is now **intent-to-treat (ITT) over the admitted pre-treatment pair set** — no admitted pair is dropped — and the gate is a **worst-case (Manski-style) bound** instead of a filled-only median: each censored **intraday** cell is imputed to a frozen adversarial cap `IS_cap_hi`, each censored **batch** cell to a frozen favorable cap `IS_cap_lo`, so Δ = (intraday − batch) is maximised **against** PASS; **PASS requires the worst-case one-sided upper CB of Δ to stay below the +10-bps margin**. The **complete-case** Δ is reported alongside and must agree (complete-case-PASS / worst-case-FAIL → NOT a PASS — exactly the cherry-pick the gate catches), and a frozen **`c_max` = 10%** max-censoring precondition makes the IS gate non-evaluable (operations-only) above it. `IS_cap_hi` (95th pct) / `IS_cap_lo` (5th pct of the pre-canary §3 calibration IS distribution) / `c_max` are added to the frozen synthetic-baseline pre-registration (new artifact §7) as researcher DOF — frozen before canary, never from canary data, covered by `synthetic_baseline_prereg_sha`. §9.1 / §9.3 reference the new gate; the §19 deferred-footnote is flipped to RESOLVED and a §20 round-4-censoring map is added.
