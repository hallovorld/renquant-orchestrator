# S-TC CLI entrypoint

DATE: 2026-07-04

## What

Added `rq-tc` CLI entrypoint for TC measurement. Outputs human-readable
summary (default) or JSON (`--json`). Exit code: 0 = TC >= 0, 1 = TC < 0,
2 = no data.

Also replaced hardcoded DEFAULT_DB path with `default_data_root()`.

## Round 2 (2026-07-04): scoped down per codex review

Codex held this PR: it cherry-picked the QP-status diagnostic (`by_qp_status`
in `tc_summary()`, `qp_infeasible` in `compute_tc_per_run()`, and the
"By QP status (ROOT CAUSE diagnostic)" section in the CLI's rendered
output) from the still-fragile `#308` taxonomy, which collapses every
non-infeasible `qp_status` value into a claimed "optimal" label. Landing
that in an ops-facing CLI would let the unproven taxonomy start being
treated as fact.

Checked `#308`'s state before choosing a fix: still open, unmerged
(`gh pr view 308 --json headRefOid,mergedAt` → `mergedAt: null`). Rebasing
onto a corrected diagnostic (codex's option (a)) would mean depending on
an unmerged sibling branch, so took option (b) instead: removed the
QP-status/infeasibility breakdown entirely from this PR.

Removed:
- `qp_status` column from the `_load_candidate_weights()` query and the
  `qp_status`/`qp_infeasible` fields from `compute_tc_per_run()`'s output.
- `by_qp_status` from `tc_summary()`.
- The "By QP status (ROOT CAUSE diagnostic)" section from `_render_summary()`.
- The two tests that exercised the removed fields
  (`test_tc_by_qp_status`, `test_tc_qp_infeasible_flag`).

Kept: the CLI entrypoint, `default_data_root()` DB-path fix, the neutral
TC measurement (mean/median/std/range/rank correlation), the by-regime
breakdown, and the loss decomposition (`tc_decomposition` — blocked vs.
shrunken vs. expanded candidates), none of which depend on `qp_status` or
assert anything about solver feasibility. Verified the CLI's rendered
output no longer contains any "root cause" / infeasibility claim
(`grep -in "root cause\|qp_status\|qp_infeasible\|infeasible"` on the
module now returns nothing) and that the remaining sections stand alone
as a coherent report. Full test suite green after the removal.

The infeasible-vs-optimal breakdown is deferred to a later PR, to land
once `#308`'s corrected taxonomy has merged and had a chance to prove out
on real data.
