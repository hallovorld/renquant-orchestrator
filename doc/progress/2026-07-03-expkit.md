# expkit — reusable pre-registered experiment framework (sprint centerpiece)

STATUS:   implemented; library + tests + migration proof, docs only otherwise. No production
          behavior change. REVISION: r1.
WHAT:     `src/renquant_orchestrator/expkit/` — the 07-02/03 measurement burst's proven
          machinery (C2/C3/C4, M3, M8, S9, RS-5, D3 + verifications, ~8,800 script lines,
          6+ re-implementations) consolidated into one importable, tested library so every
          future experiment is a FrozenSpec + callbacks:
          prereg (freeze-first spec hash + git-history committed-before-results check),
          evaluation (per-date Spearman IC, both frozen placebo-shift conventions, paired
          deltas, M4-b matched-admission solver), stats (carried-mask block bootstrap,
          AUTOMATIC small-n refusal -> exact tail masses + mandatory-null-control stamp per
          the V3 method note, multi-seed unanimity per #264), controls (mandatory positive
          plant + true null; runner refuses verdicts without both), evidence (S-REL §3.1
          content-hash manifests + verifying loader), verdict (GO/KILL/NULL/INCONCLUSIVE/
          NON-VOTING with mechanical MDE-vs-observed power call; VERDICTS.md row emitter),
          runner (freeze -> controls -> evaluation -> stats -> verdict -> evidence bundle;
          --dry-run).
          MIGRATION PROOF: C2 quality (#275) re-expressed as `plugins/c2_quality.py`;
          regression fixture reproduces the committed c2_results.json gate values (all-seed
          bootstrap bounds to ~1e-13, mechanical INCONCLUSIVE, governing NON-VOTING, freeze
          check passes on real history).
WHY:      S-REL §3.3 deferred tooling until V1–V3 resolved; they have (UPHELD/UPHELD/
          WEAKENED — machinery fragility is live). Every new 500-line harness re-pays the
          λ-round-1-class implementation-bug risk; the library pays it once, under tests.
FOUND:    writing the tests surfaced a real orientation bug in the small-n exact branch —
          the two exact methods' raw tail masses have opposite GO/KILL orientation; a
          planted positive under the sign-test fallback would have read KILL. Fixed with
          explicit oriented `go_evidence_p`/`kill_evidence_p` fields + regression test.
TESTS:    95 new (8 modules: prereg 14, evaluation 13, stats 16, controls 11, evidence 9,
          verdict 11, runner 12, C2 regression 9 — incl. small-n refusal, controls-gate
          enforcement, manifest round-trip/drift-refusal, freeze-first violations on temp
          repos). Full suite 1566 passed / 3 skipped (baseline 1471/3).
BOUNDS:   not a mass migration (merged scripts stay generators of record); no production
          import; memo-compliance CI still deferred per S-REL §3.3.
NEXT:     next verdict-producing experiment (C1 close-out, any V4–V7 promotion, M-SIG
          successor tests) plugs in as spec + callbacks; migrate old scripts only when
          touched.
DOCS:     doc/design/2026-07-03-expkit.md (design + usage sketch).

ROUND 2 (codex review): CI failure was a shallow-clone artifact, not a real freeze-first
          violation — `ci.yml`'s `test` job checkout had no `fetch-depth` (defaults to a
          depth-1 shallow clone), so `_first_commit_adding`'s `git log --diff-filter=A`
          could only ever see the single shallow tip commit for every file, making the
          spec and results look like they landed in the same commit. Reproduced locally
          (`git clone --depth 1` shows both files at commit 006dd2b; `git fetch
          --unshallow` then correctly shows spec at d860820, results at dbeb26b — genuinely
          different commits, spec strictly first). Fixed by adding `fetch-depth: 0` to the
          checkout step (matches what require-progress-doc.yml already does). Not a change
          to this PR's own code — a pre-existing CI-config gap this PR's new test was the
          first to expose.
          Scope concern (abstraction size for a substrate-constrained workflow): not
          re-scoping unilaterally. This is a consolidation of ALREADY-EXISTING duplicated
          logic (~8,800 script lines / 6+ re-implementations across the sprint), not new
          speculative capability — it explicitly does not touch production (BOUNDS: no
          production import, not a mass migration) and already caught one real bug during
          test-writing (the small-n exact-branch orientation bug, FOUND above). Flagging
          for codex to weigh in on whether that reasoning changes the scope call, rather
          than cutting library surface without full design context.
