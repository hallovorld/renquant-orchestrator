## What

New module `src/renquant_orchestrator/m6_restamp.py` (~200 lines) implementing
the orchestrator-owned v1 fingerprint re-stamp tool for the M6 migration.

Functions:
- `find_model_metadata(artifacts_dir)` -- recursive walk for model_metadata.json
- `load_metadata(path)` -- JSON load + validation
- `compute_v1_fingerprint(metadata)` -- delegates to renquant_common.model_fingerprint.model_content_sha256
- `restamp_metadata(path, dry_run=True)` -- compute v1, write stamp fields, preserve legacy
- `verify_roundtrip(path)` -- re-read + verify via renquant_common.model_fingerprint.verify
- `main(argv)` CLI -- argparse with --artifacts-dir, --dry-run (default), --apply, --verify, --output-report

## Design properties

- Hash logic is IMPORTS ONLY from renquant_common.model_fingerprint (the triple-impl lesson)
- dry_run=True is the default -- never writes production artifacts without explicit opt-in
- Preserves prior legacy hash as model_content_fingerprint_legacy_081 (audit/rollback metadata)
- Writes restamp_provenance with tool identity, timestamp, schema version, prior state
- Writes fingerprint_schema_version: 1 alongside the v1 hash
- Fail-closed: UnclassifiedKeyError / NonFiniteValueError / FingerprintError all abort without writing

## Tests

- CLI: dry-run, apply, verify, report, error codes, custom filename (7)

All 38 pass. Related fingerprint tests (check_model_bundle_consistency, prestamp_legacy, fingerprint_census) also pass (70 total).
